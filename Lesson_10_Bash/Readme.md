# Занятие 10. Bash

## Домашнее задание
Пишем скрипт.

**Цель:**
Написать bash-скрипт, который ежечасно формирует и отправляет на email отчёт о работе веб-сервера.

Что нужно сделать?

Написать скрипт для CRON, который раз в час формирует отчёт и отправляет его на заданную почту.

Отчёт должен содержать:
- IP-адреса с наибольшим числом запросов (с момента последнего запуска);
- Запрашиваемые URL с наибольшим числом запросов (с момента последнего запуска);
- HTTP-коды ответов с указанием их количества (с момента последнего запуска);
- Ошибки веб-сервера/приложения (с момента последнего запуска).

Скрипт должен предотвращать одновременный запуск нескольких копий, до его завершения.

В письме должен быть прописан обрабатываемый временной диапазон.

Декомпозиция задачи:
- Отобрать IP-адреса, посчитать сколько раз они повторяются, отсортировать по убыванию, оставить для отчета топ 10 адресов.
- Вычленить URL, посчитать сколько раз они повторяются, отсортировать по убыванию, оставить для отчета топ 10 URL.
- Вычленить HTTP-коды ответов, посчитать сколько раз они повторяются, отсортировать по убыванию, оставить для отчета топ 10 HTTP-кодов ответов.
- Вычленить ошибки - про какие ошибки идет речь? Целиком строки с ошибками в отчет пихнуть?.

А сколько строк лога нужно проверить (должно попасть в выборку)?
Отчет - один файл или на каждый пункт отдельный? Каждый час новый отчет за прошедший час.
Как я могу продолжить анализ лога с места остановки?

План решения задачи:
Построение обвязки
1. Скрипт-болванка (hello world), который крон будет запускать раз в час (минуту для теста) с предотвращением запуска копии.
2. Отправку файла по почте реализовать непосредственно в скрипте?

Предотвращение одновременного запуска нескольких копий скрипта до его завершения.

Вариант 1: Блокировка всего скрипта (Рекомендуемый) Добавьте этот блок в самое начало вашего скрипта:
```
#!/bin/bash

# Имя файла блокировки
LOCKFILE="/var/lock/my_script.lock"

# Проверяем и создаем блокировку
exec 9> "$LOCKFILE"
if ! flock -n 9; then
    echo "Ошибка: Скрипт уже запущен." >&2
    exit 1
fi

echo "Скрипт запущен и выполняется..."
# --- Основной код скрипта ---
sleep 10
echo "Скрипт завершил работу."
```

Симитировать длительность выполнения скрипта (10-15 секунд например sleep). А что если выполнение скрипта зависнет, пройдет час и новый отчет не сможет сформироваться.
Встроить эту проверку не в крон, а в сам скрипт? - Да. То есть скрипт, перед тем как приступить к выполнению, должен проверить не выполняется ли уже такой скрипт. Если выполняется, то выдать ошибку.
А где я увижу ошибку выполнения flock, если скрипт еще работает? В варианте от ИИ резульат работы проверка выводится в консоль. Предпочтительно выводить ошибки в файл журнала, а не в консоль.
